# Documentation Generation Rules

When creating or modifying documentation files:

1. **Format**: Output valid markdown. Validate syntax before finalizing.
2. **Structure**: Use hierarchical headers (h2-h4). Include emoji prefixes for major sections.
3. **Diagrams**: Use Mermaid format exclusively. ALWAYS invoke mermaid-diagram-validator tool before saving diagram content.
4. **Consistency**: Read 2-3 existing docs files to match established structure patterns.
5. **Clarity**: Support both scanning (clear headers, lists) and detailed reading (comprehensive explanations).
